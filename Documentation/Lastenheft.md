Funktionale Anforderungen:

    1. FALLS Parkplätze verfügbar sind, MUSS DAS SYSTEM Parkhaus dem Kunden DIE MÖGLICHKEIT BIETEN, zu parken.
    2. FALLS keine Parkplätze verfügbar sind, MUSS DAS SYSTEM Parkhaus FÄHIG SEIN, dem Kunden dies mitzuteilen.
    3. FALLS ein Kunde rausfahren will MUSS DAS SYSTEM Parkhaus GEWÄHRLEISTEN, dass der Kunde bezahlt hat. 
    4. DAS SYSTEM Parkhaus MUSS FÄHIG SEIN, die Anzahl und Positionen der freien Parkplätze zu kennen.
    5. DAS SYSTEM Parkhaus SOLLTE FÄHIG SEIN, Daten wie Kunden, Parkzeiten und Gebühren zu speichern und abzurufen.
	6. DAS SYSTEM Parkhaus SOLLTE FÄHIG SEIN, die Öffnungszeiten festzulegen.

Nicht-funktionale Anforderungen:

    DAS SYSTEM Parkhaus SOLLTE dem Anwender DIE MÖGLICHKEIT BIETEN, ein GUI zur Bedienung zu nutzen.
